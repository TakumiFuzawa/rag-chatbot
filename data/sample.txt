AI-900 Microsoft Azure AI Fundamentals 試験対策

■ 第1章：AIの基本概念

1-1. 人工知能（AI）とは
人工知能とは、人間の知的行動をコンピューターで再現する技術です。
AIは大量のデータからパターンを学習し、予測や判断を行います。
主な活用例として、画像認識、音声認識、自然言語処理などがあります。

1-2. 機械学習とは
機械学習はAIの一分野で、データからアルゴリズムが自動的に学習する技術です。
明示的にプログラムしなくても、データから規則性を見つけ出すことができます。
機械学習には教師あり学習、教師なし学習、強化学習の3種類があります。

1-3. 教師あり学習
教師あり学習は、正解ラベル付きのデータを使って学習する方法です。
入力データと正解データのペアを大量に学習させることでモデルを構築します。
分類（カテゴリ予測）と回帰（数値予測）が代表的なタスクです。
例：メールのスパム判定、住宅価格の予測

1-4. 教師なし学習
教師なし学習は、正解ラベルなしのデータからパターンを見つける方法です。
データのグループ分け（クラスタリング）や次元削減に使われます。
例：顧客のグループ分け、異常検知

1-5. 深層学習（ディープラーニング）
深層学習は、人間の脳神経回路を模倣したニューラルネットワークを使う技術です。
画像認識、音声認識、自然言語処理で高い精度を発揮します。
大量のデータと高い計算能力が必要です。

■ 第2章：Azure AIサービス

2-1. Azure OpenAI Service
Azure OpenAIは、OpenAIの大規模言語モデルをAzure上で利用できるサービスです。
GPT-4、GPT-4o-miniなどのモデルでテキスト生成、要約、翻訳ができます。
企業向けのセキュリティとコンプライアンスが確保されています。
プロンプトエンジニアリングにより、様々なタスクに対応できます。

2-2. Azure AI Search
Azure AI Searchは、AIを活用したクラウド検索サービスです。
ベクトル検索とセマンティック検索に対応しています。
RAGアーキテクチャの中核コンポーネントとして使われます。
大量のドキュメントから関連情報を素早く検索できます。

2-3. Azure AI Vision
Azure AI Visionは、画像や動画を分析するサービスです。
物体検出、顔認識、OCR（文字認識）などの機能があります。
事前学習済みモデルを使うことで、すぐに利用開始できます。
カスタムモデルの学習にも対応しています。

2-4. Azure AI Language
Azure AI Languageは、テキスト分析を行うサービスです。
感情分析、キーフレーズ抽出、言語検出などの機能があります。
固有表現認識（NER）で人名、地名、組織名などを抽出できます。
質問応答システムの構築にも利用できます。

2-5. Azure AI Speech
Azure AI Speechは、音声認識と音声合成のサービスです。
音声をテキストに変換（Speech to Text）できます。
テキストを音声に変換（Text to Speech）もできます。
リアルタイム翻訳にも対応しています。

2-6. Azure AI Document Intelligence
Azure AI Document Intelligenceは、文書からデータを抽出するサービスです。
PDF、画像、Word文書などから情報を自動抽出できます。
請求書、レシート、契約書など様々な文書に対応しています。
OCR技術とAIを組み合わせて高精度な文字認識を実現しています。

2-7. Azure Machine Learning
Azure Machine Learningは、機械学習モデルの開発・デプロイ環境です。
データの準備からモデルの学習、デプロイまで一元管理できます。
AutoMLを使えばコードなしでモデルを自動構築できます。
MLOpsのワークフロー管理にも対応しています。

■ 第3章：RAGアーキテクチャ

3-1. RAGとは
RAGとはRetrieval Augmented Generationの略です。
検索（Retrieval）と生成（Generation）を組み合わせた技術です。
ドキュメントを検索して関連情報を取得し、AIが回答を生成します。
社内文書や専門資料に基づいた正確な回答が可能になります。

3-2. RAGの仕組み
ステップ1：ドキュメントをベクトル化してデータベースに保存します。
ステップ2：ユーザーの質問をベクトル化します。
ステップ3：類似度の高いドキュメントを検索します。
ステップ4：検索結果を元にAIが回答を生成します。

3-3. ベクトル検索
ベクトル検索は、テキストを数値の配列（ベクトル）に変換して検索する技術です。
意味的に近いテキストは、ベクトル空間でも近い位置に配置されます。
キーワード検索より文脈を考慮した検索が可能です。
Azure AI Searchはベクトル検索に対応しています。

■ 第4章：責任あるAI

4-1. 責任あるAIの原則
Microsoftは責任あるAIのための6つの原則を定めています。
公平性：AIはすべての人を公平に扱う必要があります。
信頼性と安全性：AIは安全で信頼できる動作をする必要があります。
プライバシーとセキュリティ：データは安全に保護される必要があります。
包括性：AIはすべての人が利用できる必要があります。
透明性：AIの動作は説明可能である必要があります。
説明責任：AIの開発者と利用者は結果に責任を持つ必要があります。

4-2. AIの公平性
AIモデルは学習データのバイアスを引き継ぐことがあります。
性別、人種、年齢などによる差別が生じないよう注意が必要です。
Azure Machine LearningにはFairlearnという公平性評価ツールがあります。

4-3. AIの透明性
AIの判断根拠を説明できることが重要です。
説明可能なAI（XAI）の技術が注目されています。
Azure Machine LearningではモデルのShap値を確認できます。
